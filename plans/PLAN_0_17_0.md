## Milestone Goals

- **B-4:** Cost-Based Refresh Strategy Selection — replace the heuristic cost
  model with a statistics-driven planner that picks FULL vs. DIFFERENTIAL at
  runtime based on buffer size, row count, and DML patterns.
- **A-2-COL:** Columnar Change Tracking — propagate `changed_cols` bitmasks
  end-to-end so unchanged projections short-circuit without touching data.
- **A2:** Transactional IVM Phase 4 — A2-PS (`use_prepared_statements`) ✅
  already shipped; A2-ENR (ENR transition tables) and A2-CTR (C-level
  triggers) 🚫 deferred post-1.0 (raw `pg_sys` FFI risk).
- **A8:** `ROWS FROM()` multi-SRF support.
- **SQLANCER:** SQLancer fuzzing integration for parser and refresh correctness.
- **C-2:** Incremental DAG Rebuild — avoid full DAG recomputation on DDL.
- **UNSAFE-R1/R2:** Unsafe block reduction Phase 6.
- **API-MOD:** `api.rs` modularization into logical sub-modules.
- **MIG-IVM:** pg_ivm migration guide.
- **RUNBOOK:** Failure mode runbook.
- **PLAYGROUND:** Docker quickstart playground.
- **DOC-POLISH:** Documentation polish pass.
- **TG2-MERGE / TG2-CANCEL:** Deferred test coverage items from v0.16.0.
