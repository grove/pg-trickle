---

## Milestone Goals

- Downstream CDC publication so stream table changes can drive Kafka, Debezium, and event-sourcing pipelines
- Minimal-viable in-database parallel refresh worker pool for throughput gains
- Predictive cost model for adaptive refresh (linear regression over refresh history)
- SLA-driven tier auto-assignment (no manual tier tuning)

---

## Recommended Implementation Order

### Phase 1 — CDC Publication (CDC-PUB)
1. CDC-PUB-1: `stream_table_to_publication()` SQL function
2. CDC-PUB-2: Lifecycle management (drop, auto-drop, recreation)
3. CDC-PUB-3: `downstream_publication` column in `pg_stat_stream_tables`
4. CDC-PUB-4: E2E tests
5. CDC-PUB-5: Documentation

### Phase 2 — Parallel Refresh (PAR)
1. PAR-1: Coordinator / worker process split
2. PAR-2: `pg_trickle.max_parallel_workers` GUC
3. PAR-3: DAG level extraction
4. PAR-4: Worker crash recovery
5. PAR-5: E2E tests

### Phase 3 — Predictive Cost Model (PRED)
1. PRED-1: Linear regression forecaster
2. PRED-2: Pre-emptive FULL switch
3. PRED-3: Cold-start fallback
4. PRED-4: E2E test + proptest

### Phase 4 — SLA-Driven Tier (SLA)
1. SLA-1: `sla` parameter on create/alter
2. SLA-2: Initial tier assignment
3. SLA-3: Dynamic re-assignment
4. SLA-4: E2E test
