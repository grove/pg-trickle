# PLAN_0_19_0.md — v0.19.0 Implementation Order

**Milestone:** v0.19.0 — Production Gap Closure & Distribution
**Status:** 🚧 In progress
**Last updated:** 2026-04-13


This document defines the recommended implementation order for all v0.19.0
roadmap items. See [ROADMAP.md §v0.19.0](../ROADMAP.md#v0190--production-gap-closure--distribution)
for the full feature descriptions and effort estimates.

---

## Milestone Goals

- **CORR-1:** Remove the unsafe `delete_insert` merge strategy (P0).
- **CORR-2:** Fix JOIN delta R₀ — simultaneous key-change + right-side delete
  leaves a stale row in the stream table (P1 correctness gap from Phase 7).
- **CORR-3/4:** Extend DDL event classifier to track `ALTER TYPE`,
  `ALTER DOMAIN`, and `ALTER POLICY` as stream-table-invalidating events.
- **CORR-5:** Fix keyless content-hash collision for identical-content rows.
- **CORR-6:** Harden guarded `.unwrap()` calls in DVM operators.
- **CORR-7:** TRUNCATE + INSERT CDC ordering guarantee within a transaction.
- **CORR-8:** NULL join-key delta handling for INNER/OUTER joins.
- **SEC-1:** Add ownership checks to `drop_stream_table` and
  `alter_stream_table` — prevent non-owner roles from modifying stream tables.
- **SEC-2:** SQL injection audit for dynamic refresh SQL (`format!()` sites).
- **STAB-1:** PgBouncer transaction-mode compatibility guard via new
  `pg_trickle.connection_pooler_mode` GUC.
- **STAB-2:** Read-replica / hot-standby safety guard — worker exits cleanly
  instead of crash-looping when started on a replica.
- **STAB-3:** Elevate Semgrep to blocking in CI.
- **STAB-4:** `auto_backoff` GUC — double effective interval after 3
  consecutive falling-behind alerts.
- **STAB-5:** Harden `unwrap()` in scheduler hot path (pairs with PERF-5).
- **STAB-6:** Parallel worker crash recovery sweep (orphaned locks, stuck
  REFRESHING state).
- **STAB-7:** Extension version mismatch detection at load (`.so` vs SQL).
- **PERF-1/2:** Fix WAL decoder bugs (`old_*` NULL on UPDATE, naive action
  string parsing) before the WAL path is promoted to production-ready.
- **PERF-4:** Add catalog indexes on `pgt_relid` and `pgt_dependencies(pgt_id)`.
- **PERF-5:** Eliminate O(n²) `units().find()` in scheduler dispatch via HashMap.
- **PERF-6:** Batch `has_table_source_changes()` into single SPI query.
- **PERF-7:** Delta branch pruning for zero-change sources in multi-JOIN.
- **PERF-8:** Index-aware MERGE path selection for small deltas.
- **UX-1:** Flip PGXN `release_status` to `"stable"`.
- **UX-2:** Automate Docker Hub release pipeline.
- **UX-3:** apt/rpm packaging via PGDG.
- **UX-4:** Connection pooler compatibility guide in `docs/PRE_DEPLOYMENT.md`.
- **UX-6:** Change `drop_stream_table` cascade default to `false`.
- **UX-7:** Resolve OIDs to table names in error messages.
- **UX-8:** Emit NOTICE when `refresh_stream_table` is skipped.
- **UX-9:** Fix CONFIGURATION.md TOC gaps.
- **UX-10:** TUI per-table refresh latency sparkline.
- **UX-11:** `pgtrickle.version()` diagnostic function.
- **DB-1:** Fix duplicate `'DIFFERENTIAL'` in two CHECK constraints (blocks 1.0).
- **DB-2:** Add `ON DELETE CASCADE` FK on `pgt_refresh_history.pgt_id`.
- **DB-3:** Add `pgtrickle.pgt_schema_version` version tracking table.
- **DB-4:** Rename `pgtrickle_refresh` NOTIFY channel → `pg_trickle_refresh`.
- **DB-5:** `pg_trickle.history_retention_days` GUC + scheduler daily cleanup.
- **DB-6:** Document public API stability contract in `docs/SQL_REFERENCE.md`.
- **DB-7:** Populate `sql/pg_trickle--0.18.0--0.19.0.sql` with DB-1 through DB-4 migrations.
- **DB-8:** Validate orphan cleanup in `drop_stream_table` for `pgt_change_tracking`.
- **DB-9:** `pgtrickle.migrate()` utility function.
- **SCAL-3:** CNPG operational runbook in `docs/SCALING.md`.
- **SCAL-4:** Partitioned source table impact assessment (time-boxed spike).
- **TEST-9:** Extract unit-testable logic from E2E-only refresh/scheduler paths.
- **TEST-10:** TPC-H scale factor coverage (SF-1, SF-10) in nightly CI.

stability fixes, distribution, and documentation.
