This document defines the recommended implementation order for all v0.28.0

## Milestone Goals

Build a complete, production-grade solution for the two most common
event-driven integration patterns:

- **Part A (Essential):** Transactional Outbox + Transactional Inbox
- **Part B (Advanced):** Consumer Groups + Ordered Inbox Processing

---

## Recommended Implementation Order

### Phase A-SHARED: Catalog Infrastructure (Day 1)

1. Version bump: `0.27.0` → `0.28.0`
2. Error variants: `OutboxAlreadyEnabled`, `OutboxNotEnabled`, `OutboxRequiresNotImmediateMode`,
   `InboxAlreadyExists`, `InboxNotFound`, `InboxTableNotFound`, `InboxColumnMissing`,
   `InboxOrderingPriorityConflict`, `ConsumerGroupAlreadyExists`, `ConsumerGroupNotFound`
3. GUC additions (12 outbox + 6 inbox)
4. Catalog tables: `pgt_outbox_config`, `pgt_inbox_config`
5. SQL upgrade script: `sql/pg_trickle--0.27.0--0.28.0.sql`

### Phase A-OUTBOX: Outbox Helper (Days 1–5)

6. `enable_outbox(name, retention_hours)` / `disable_outbox(name, if_exists)`
7. Outbox table DDL: `pgtrickle.outbox_<st>`, `pgtrickle.pgt_outbox_latest_<st>`, `pgtrickle.outbox_delta_rows_<st>`
8. Refresh-path integration (post-MERGE outbox INSERT, claim-check routing, NOTIFY)
9. Versioned payload format (inline + claim-check)
10. Retention drain scheduler step
11. Lifecycle & cascade (`drop_stream_table`, `outbox_status`, `outbox_rows_consumed`)

### Phase A-INBOX: Inbox Helper (Days 5–11)

12. `create_inbox()` + `pgt_inbox_config` catalog
13. Inbox table DDL + indexes + autovacuum
14. Auto-created stream tables (pending, DLQ, stats)
15. `enable_inbox_tracking()` for existing tables
16. DLQ alert mechanism (post-refresh hook)
17. `inbox_health()` + `inbox_status()`
18. Retention drain + `replay_inbox_messages()`
19. `drop_inbox()` lifecycle

### Phase A-TEST: Part A Tests (Days 11–13)

20. Unit tests: outbox enable/disable/naming/cascade
21. Integration tests: outbox end-to-end, claim-check boundary, relay cursor
22. Integration tests: inbox lifecycle, DLQ routing, retention
23. SHARED-3: full pipeline E2E test

### Phase A-DOC: Part A Documentation (Days 13–14)

24. SQL_REFERENCE.md: outbox API, payload formats, NOTIFY channel
25. SQL_REFERENCE.md: inbox API
26. CONFIGURATION.md: 12+6 GUCs
27. PATTERNS.md: event-driven integration chapter
28. Reference examples: outbox_relay.py, inbox_processor.py

### Phase B-OUTBOX: Consumer Groups (Days 14–22)

29. `pgt_consumer_groups`, `pgt_consumer_offsets`, `pgt_consumer_leases` catalog tables
30. `pgt_consumer_claim_check_acks` catalog table
31. `create_consumer_group()` / `drop_consumer_group()`
32. `poll_outbox()` with visibility timeout and lease management
33. `extend_lease()` lease renewal function
34. `commit_offset()` + `seek_offset()`
35. Heartbeat + liveness (`consumer_heartbeat()`, `consumer_lag()`)
36. Monitoring stream tables (`pgt_consumer_status`, `pgt_consumer_group_lag`, `pgt_consumer_active_leases`)
37. Dead consumer auto-cleanup scheduler step
38. Retention safety guard (consumer-aware drain)

### Phase B-INBOX: Ordered Processing (Days 22–28)

39. `enable_inbox_ordering()` + `next_<inbox>` aggregate-ordered stream table
40. `gaps_<inbox>` gap detection stream table + alert
41. `enable_inbox_priority()` + tier-based stream tables
42. `inbox_is_my_partition()` worker affinity helper

### Phase B-TEST: Part B Tests (Days 28–31)

43. Consumer group integration tests
44. Ordered inbox integration tests
45. Multi-relay E2E test (SHARED-B3)

### Phase B-DOC: Part B Documentation (Days 31–33)

46. SQL_REFERENCE.md: consumer group API, ordering API
47. CONFIGURATION.md: Part B GUCs
48. PATTERNS.md: advanced sections
49. Reference relay implementations
50. dbt adapter updates (SHARED-B4)
