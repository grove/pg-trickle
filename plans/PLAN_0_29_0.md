## Milestone Goals

- Ship `pgtrickle-relay` — a standalone bidirectional Rust CLI binary that
  bridges pg-trickle outboxes and inboxes with popular messaging systems.
- **Forward mode**: poll outbox tables → publish to external sinks (NATS,
  Kafka, HTTP webhook, Redis, SQS, RabbitMQ, PostgreSQL inbox, stdout/file).
- **Reverse mode**: consume from external sources → write to pg-trickle inbox.
- SQL catalog schema (`relay_outbox_config`, `relay_inbox_config`,
  `relay_consumer_offsets`) with 7 SQL API functions.
- Observable: Prometheus metrics at `:9090/metrics`, health at `/health`.
- Configurable via PostgreSQL catalog tables (hot-reload via LISTEN/NOTIFY).
- Horizontally scalable via PostgreSQL advisory locks (no external coordinator).
