# PLAN_0_15_0.md ‚Äî v0.15.0 Implementation Order

**Milestone:** v0.15.0 ‚Äî External Test Suites & Integration
**Status:** üîÑ In Progress
**Last updated:** 2026-04-04

This document defines the recommended implementation order for all v0.15.0
roadmap items.

Sequencing is driven by:

1. **Dependency order** ‚Äî G13-PRF (parser modularization) is a prerequisite
   for PG backward compatibility in v0.16.0; front-load it if possible.
2. **Risk front-loading** ‚Äî Phase 1 (TS3 + I2) ships independently
   of the two large structural items (G13-PRF, WM-7).
3. **Independent features early** ‚Äî TS3, I2, I3, G15-BC, PH-E1, PH-D2, TRUNC-1,
   VOL-1, EC-01, STST-3, CIRC-IMM, E4, and E5 are structurally independent; any
   can slip without blocking the others.
4. **Large refactors later** ‚Äî G13-PRF is ~3‚Äì4 weeks of careful work; it
   should not block the release if deadlines tighten.
5. **Research spike early** ‚Äî G14-SHC-SPIKE informs v0.16.0 planning; do it
   early so the RFC is available before parser modularization completes.

- RFC written: `plans/performance/RFC_SHARED_TEMPLATE_CACHE.md`

- Data structures: `CacheHeader` (64 B) + `SlotIndex[max_entries]` (32 B each) + payload area (configurable, default 8 MB)
- Invalidation: integrates with existing `CACHE_GENERATION` counter + per-ST `defining_query_hash` validation
- Locking: LWLock with shared reads (cache hit ~2‚Äì5 Œºs) and exclusive writes (cache populate; serialization <10 Œºs per slot)
- Size budget: ~4 MB for 1024 entries (vs ~200 MB with 100 backends √ó thread-local caches)
- Go/no-go criteria defined: ‚â•90% cold-start reduction, <1% steady-state regression, <16 MB for 500 STs
- Alternatives considered: prewarming, ShmemInitHash, mmap, external cache ‚Äî all rejected with rationale
- Full implementation plan: Phase A (infrastructure), B (integration), C (benchmarking) for v0.16.0
