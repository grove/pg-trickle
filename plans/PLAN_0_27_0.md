# PLAN_0_27_0.md — v0.27.0 Implementation Order

**Milestone:** v0.27.0 — Operability, Observability & DR
**Status:** 🚧 In progress
**Last updated:** 2026-04-21

This document defines the recommended implementation order for all v0.27.0
roadmap items. See [ROADMAP.md §v0.27.0](../ROADMAP.md#v0270--operability-observability--dr)
for the full feature descriptions and effort estimates.

---

## Milestone Goals

- Ship a snapshot & PITR API so replicas can bootstrap from point-in-time exports
- Add a predictive maintenance window planner that turns cost-model history into schedule recommendations
- Expose cluster-wide worker allocation from the postmaster; add per-DB Prometheus labels
- Harden the OpenMetrics endpoint: conformance test, port-conflict handling, malformed-HTTP 400
- Upgrade pgrx 0.17.0 → 0.18.0 and validate the full test suite

---

## Recommended Implementation Order

### Phase 1 — Dependency Upgrade (DEP-1/2)

Upgrade pgrx first to avoid re-compiling twice during feature work.

### Phase 2 — Snapshot/PITR (SNAP-1/2/3/4/5)

New `pgtrickle.snapshot_stream_table()`, `restore_from_snapshot()`,
`list_snapshots()`, `drop_snapshot()` SQL functions plus error variants,
integration tests, and documentation.

### Phase 3 — Predictive Planner (PLAN-1/2/3/4)

New `pgtrickle.recommend_schedule()` and `pgtrickle.schedule_recommendations()`
SQL functions, spike-forecast alert, two new GUCs, unit tests, documentation.

### Phase 4 — Cluster Observability (CLUS-1/2/3/4)

New `pgtrickle.cluster_worker_summary()` SQL function, per-DB Prometheus label
injection, new multi-tenant integration doc, SCALING.md update.

### Phase 5 — Metrics Hardening (METR-1/2/3/4)

OpenMetrics conformance unit test, port-conflict and timeout unit tests,
`pgtrickle.metrics_summary()` SQL function, malformed-HTTP 400 handler.

### Phase 6 — Integration, Upgrade Script, Version Bump

Upgrade migration SQL, archive SQL, version sync, CHANGELOG.md entry.
