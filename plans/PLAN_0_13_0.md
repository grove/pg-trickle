# PLAN_0_13_0.md — v0.13.0 Implementation Order

**Milestone:** v0.13.0 — Scalability Foundations, Partitioning Enhancements, MERGE Profiling & Multi-Tenant Scheduling
**Status:** 🚧 In Development
**Last updated:** 2026-03-31

This document defines the recommended implementation order for all v0.13.0
roadmap items. The scope is the set of phases deferred from v0.12.0 (Phases
5–8) plus three new phases added after the v0.12.0 scope was locked.

Sequencing is driven by:

1. **Dependency order** — A-2 (columnar tracking) must precede D-4 (shared
   buffers); TPC-H benchmark harness (Phase 10) should land before the
   scalability work so that performance gains can be measured objectively.
2. **Risk front-loading** — Phase 10 (TPC-H harness) and Phase 11 (SQL
   cleanup) are cheap, zero-risk, and provide immediate value; ship them
   in Phases 1–2 so the rest of the milestone has a measurement baseline.
3. **Correctness-sensitive phases last** — D-4 multi-frontier cleanup is
   the hardest correctness hazard in this release. Gate it behind A-2 and
   behind a dedicated property-based test harness.
4. **Low effort, high value early** — MERGE profiling, dbt macro updates,
   and SQL cleanup ship early; they are independent of the heavy structural
   work and de-risk the milestone exit criteria.

## Priority Tiers (Scope Risk)

Total estimated scope is ~15–23 weeks across all ten phases. If the
milestone runs long, phases are deferrable in the order below. A release
can be cut with only the P0 phases complete.

| Tier | Phases | Contents | Action if time is short |
|------|--------|----------|-------------------------|
| **P0 — Must ship** | 1, 6, 7 | TPC-H harness, A-2 columnar tracking, D-4 shared buffers | Block release until done |
| **P1 — Target** | 3, 5, 8, 9, 10 | MERGE profiling, partitioning enhancements, PERF-2 auto-partitioning, multi-tenant scheduler isolation, DVM engine improvements | Ship if ready; defer individual items to v0.14.0 if not |
| **P2 — Can slip** | 2, 4 | SQL coverage cleanup, dbt macro updates | Defer entire phase to v0.14.0 as a unit |

> **Slip rule:** Never partially ship Phase 7 (D-4). It depends on Phase 6
> (A-2) and a partial shared-buffer migration is worse than no migration.
> If Phase 6 is deferred, Phase 7 must be deferred with it.

## Phase 6 — Columnar Change Tracking (A-2)

**Goal:** Eliminate wasted delta processing for wide tables. When a row is
updated but only projected columns changed, the current CDC path still emits a
full row delta. A-2 introduces a per-column bitmask that lets the Delta Scan
operator skip rows where no referenced column changed — the foundation for the
shared-buffer work in Phase 7.

> ⚠️ **Implement before Phase 7.** D-4 (shared buffers) builds on the
> per-column tracking infrastructure added here. The column-superset mode
> required by D-4 is a natural extension of the A-2 bitmask.

| Item | Description | Effort | Ref |

For value-only UPDATEs (`changed_cols & key_mask = zero`), only the 'V' row passes — carrying `(new_col - old_col)` as net correction for SUM, and contributing 0 to counts. Benefits: halved MERGE source rows for value-only updates, HOT-eligible updates (no index churn), reduced WAL. When a `SUM` aggregate's argument is `Expr::Raw` starting with `CASE` (case-insensitive, whitespace-tolerant), the function returns `false`, forcing the `GROUP_RESCAN` (EXCEPT ALL) path which correctly reads pre-change state. Q14 is unaffected — its `SUM(CASE WHEN …)` is wrapped in a multiplication/division, parsed as `AggFunc::ComplexExpression`, never reaching `is_algebraically_invertible()`. Q12 removed from `DIFFERENTIAL_SKIP_ALLOWLIST` in `tests/e2e_tpch_tests.rs`. 4 unit tests added: `test_sum_case_when_not_algebraically_invertible`, `test_sum_plain_column_still_algebraically_invertible`, `test_sum_case_when_with_leading_whitespace`, `test_count_star_still_algebraically_invertible`. This is a correctness band-aid — once DI-2 (aggregate UPDATE-split with `old_*` columns) lands, the check can be removed and SUM(CASE WHEN …) restored to the faster algebraic path.
