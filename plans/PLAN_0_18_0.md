## Milestone Goals

- **CORR-1 (CSS-3):** Cross-source snapshot consistency — LSN watermark
  enforcement so multi-source stream tables never see a split snapshot.
- **CORR-2 / TEST-1 (TPCH-BASE):** Populate TPC-H expected-output regression
  guard; turns the 22-query suite into a true correctness canary.
- **CORR-3:** NULL-safe GROUP BY elimination under deletes.
- **CORR-4 / TEST-4:** Z-set merged-delta weight accounting — property-based
  correctness tests for the merged delta engine.
- **CORR-5:** HAVING-filtered aggregate correction under group depletion.
- **STAB-1 (SAFE-1):** Eliminate all production-path `.unwrap()` calls in
  `api.rs` and `refresh.rs`.
- **STAB-2 (UNSAFE-P1):** Unsafe block reduction Phase 1 — `pg_cstr_to_str()`
  safe wrapper eliminates ~69 mechanical `unsafe` blocks.
- **STAB-3 (PH-E2):** Spill detection alerting via `AlertEvent::SpillThresholdExceeded`.
- **STAB-4:** Parallel worker orphaned resource cleanup after crash.
- **STAB-5 / TEST-6:** Upgrade migration test for the 0.16→0.17→0.18 chain.
- **PERF-1 (B3-MERGE):** Z-set multi-source delta engine — merges per-source
  delta branches into a single `GROUP BY + SUM(weight)` query.
- **PERF-2 (B-4 remainder):** Cost-based refresh strategy completion — per-ST
  cost model trained on `pgt_refresh_history`.
- **PERF-3:** Zero-change source branch elision in the merged delta builder.
- **PERF-4 (A-2-COL-1):** Columnar change tracking Phase 1 — CDC bitmask.
- **UX-1 (CACHE-OBS):** Template cache observability via `pgtrickle.cache_stats()`.
- **UX-2:** Pre-built Grafana dashboard panels.
- **TEST-2 (SQLancer):** SQLancer crash-test oracle — 10K+ fuzzed queries with
  zero panics.
- **TEST-3:** CDC edge cases — NULL PKs, composite PKs, generated columns.
- **TEST-7:** dbt integration regression coverage.

## Recommended Implementation Order

Items are sequenced to front-load P0 correctness and stability work, unblock
downstream dependencies, and batch the two schema-changing items together.

### Phase 1 — Correctness & Stability Foundation  (P0 items)

These items carry the highest risk and have no upstream dependencies. Do them
first so property-based and upgrade tests are in place before performance work
begins.

| Order | ID | Feature | Effort | Notes |
|-------|----|---------|--------|-------|
| 1 | STAB-1 | Eliminate production-path `.unwrap()` | S | No dependencies; eliminates panic risk |
| 2 | CORR-3 | NULL-safe GROUP BY under deletes | S | Simple; closes a known correctness gap |
| 3 | CORR-5 | HAVING group depletion correction | S | Similar scope to CORR-3 |
| 4 | TPCH-BASE (CORR-2/TEST-1) | Populate TPC-H expected-output baseline | XS | Needed before any perf changes land |
| 5 | TEST-3 | CDC edge cases (NULL PK, composite PK, generated cols) | M | Build test infrastructure early |

### Phase 2 — Safety Hardening

| Order | ID | Feature | Effort | Notes |
|-------|----|---------|--------|-------|
| 6 | STAB-2 | Unsafe block reduction Phase 1 | M | Mechanical; safe to run in parallel with phase 1 |
| 7 | STAB-3 | Spill detection alerting | S | Requires monitor.rs change only |

PERF-1, CORR-4, and TEST-4 form a mandatory cluster. Do not merge PERF-1
without the property tests.
