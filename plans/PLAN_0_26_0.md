## Recommended Implementation Order

### Phase 1 — Concurrency tests (Days 1–6)

1. **CONC-1**: Simultaneous ALTER + REFRESH E2E test
2. **CONC-2**: Simultaneous DROP + REFRESH E2E test
3. **CONC-3**: Parallel-worker duplicate-pick test
4. **CONC-4**: Concurrent canary promotion race test

### Phase 2 — Predictive model & SLA stability (Days 6–12)

5. **SLA-1**: Predictive cost model accuracy harness
6. **SLA-2**: SLA tier oscillation damping
7. **SLA-3**: SLA tier oscillation property test

### Phase 3 — Fuzz & scale testing (Days 12–18)

8. **FUZZ-1**: Cron parser fuzz target
9. **FUZZ-2**: GUC string→enum fuzz target
10. **FUZZ-3**: CDC trigger payload fuzz target
11. **SCALE-1**: Partition-count scale test
12. **SCALE-2**: Multi-DB worker starvation test

### Phase 4 — ARCH-1B: Refresh sub-module migration (Days 18–27)

13. **ARCH-1B-1**: Migrate orchestration to src/refresh/orchestrator.rs
14. **ARCH-1B-2**: Migrate SQL codegen to src/refresh/codegen.rs
15. **ARCH-1B-3**: Migrate MERGE execution to src/refresh/merge.rs
16. **ARCH-1B-4**: Migrate PH-D1 logic to src/refresh/phd1.rs

### Phase 5 — Error handling (Days 27–31)

17. **ERR-1**: Typed DiagnosticError variant
18. **ERR-2**: Typed PublicationError variant
19. **ERR-3**: Scheduler timestamp errors with HINT
20. **ERR-4**: Publication crash-recovery E2E test
